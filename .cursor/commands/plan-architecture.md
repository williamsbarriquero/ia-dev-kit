# Plano de Arquitetura

Gere um plano de implementação estruturado para a tarefa solicitada.

## Passos

1. Analisar requisitos e constraints do usuário.
2. Investigar o codebase (dependências, padrões existentes, pontos de integração).
3. Consultar skills: `hexagonal-architecture`, `design-patterns`, `solid-principles`.
4. Produzir plano atômico com:
   - Contexto e decisões arquiteturais
   - Tarefas numeradas com dependências
   - Riscos e mitigações
   - Critérios de verificação por tarefa
5. **Não implementar código** — handoff para `@rpe-developer.md`.

## Output

Seguir `interaction-standards.mdc` com foco em Análise e Proposta.
